# Roadmap - hardware

## Ziel

Eine nachvollziehbare Hardwaredokumentation fuer Elektrik, Verdrahtung, Aufbauablauf, Bringup und Materialstand aufbauen.

## Arbeitspakete

- [x] Hardwarebereiche und Unterordner festlegen
- [x] Aufbau- und Bringup-Bereiche als Dokumentationsstruktur einfuehren
- [x] Wearable-Sensorkonzept fuer Prototype und spaeteren Betriebstyp dokumentieren
- [x] Quellenkonvention fuer hardwarebezogene Konzeptdokumente einziehen
- [x] Elektronikbereich fuer Sender-/Receiver-ESP, Breadboardpfad, Lochrasterpfad und Pinmapping dokumentieren
- [x] Boardrevision der ESP32-S3-DevKits und RGB-LED-Pin real am Bestand verifizieren
- [x] BNO055-, Mux- und Flex-Sensor-Pinplan auf echter Bench-Hardware pruefen
- [x] Bench-Schaltplanstand fuer Controller- und Receiver-Seite dokumentieren
- [ ] reale Verkabelung gegen den Schaltplanstand abgleichen
- [x] mechanische Halterungen und Befestigungen fuer das Wearable-Konzept beschreiben
- [ ] Stueckliste fuer Prototyp und spaeteren Daueraufbau pflegen
- [ ] Inventar-, Aufbau- und Einschaltvorlagen spaeter mit realen Nachweisen fuellen
