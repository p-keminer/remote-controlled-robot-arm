# Roadmap - firmware/esp32_bridge

## Ziel

Einen stabilen, rein lesenden Debugpfad fuer Dashboard und ROS pflegen.

## Abgeschlossen

- [x] ESP-NOW-Empfang vom Controller
- [x] MQTT-Pfad zum Pi
- [x] Status-LEDs fuer WiFi, Link und MQTT
- [x] End-to-End-Verifikation mit Dashboard und ROS

## Offen

- [ ] Langzeitverhalten und Ausfallverhalten weiter dokumentieren
- [ ] OTA nur dann weiter pflegen, wenn der Debugpfad ihn wirklich noch braucht
